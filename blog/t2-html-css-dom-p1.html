Can you Think of an Analogy to Describe HTML, CSS and the DOM? How would you Describe it to your non-Technical Friend?

An analagy for HTML and CSS (and javascrpit) would be to think of it all like building a house. HTML is the foundations, the wood, and the nails which are used to build the structure of the house. CSS is the decorating of the house, so the paint color, and what your inhouse decorm will look like. then Javascript what makes everything go, like the T.V remote or your radio controls.

DOM on the otherhand is like a Tree, you can add branches, cut branches off, Grow new branches, grow or trim the leaves, or modify the whole tree entirely, by changing the trunk!

What does Boxifying mean?

Boxifying means to think of a webpage like a group of boxes stacked together. There are the main boxes such as, Head, body, footer and asie. but also these boxs contain other boxes, such as paragraphs, imgs, headers etc. boxifying means to identify the boxes of a website.

What is the Box model?

The box model is essentially a box that wraps around all HTML elements. It consists of Margins, borders, padding and the actual content.

What is Grid Based Design?

Grid systems bring structure to a website. As a tool grids are useful for organizing and presenting information in a way that creates predictable patterns for users to follow.

Whats all the Hype about Responsive Webpages?

Responsive websites are websites where the layout scales to the screensize regardless of the type of device it is viewed on. There are many types of screensizes people view websites on including desktop computer, laptops, tablets and mobile phones. A responsive website will scale itself according to the size of the screen it is being viewed on.

What is Semantic Structure?

Semantic structure is when you use "semantic" elements to format your html code. Semantic elements are elements which have a meaning, such as img, table, form, etc. As oppose to non-semantic elements such as div, span, head.
